# Hyprland Rice Reevaluation — Design

**Date:** 2026-06-14
**Author:** Tiago (cocaine)
**Status:** Approved for planning

## Overview

A cleanup + roadmap effort across the Hyprland rice at `/home/cocaine/Projects/hyprland-rice`.
The setup has drifted: stale docs, a notification daemon that lets browser
notifications hang forever, a half-migrated bar (ironbar running but unthemed and
borrowing waybar's scripts), and silently broken wallpaper switching.

Guiding principle: **fix what's silently broken first, then consolidate to one
source of truth (one bar, theming wired everywhere), then refine to workflow.**
Each front is its own phase with a checkpoint between them.

### Key architectural fact: the symlink + template pattern

- Repo dirs are **symlinks into `~/.config`**. Editing the repo *is* editing live config.
- Several live configs are **generated by flavours** from mustache templates in
  `~/.config/flavours/templates/`. The live file is regenerated on every theme
  switch. **Editing only the live file is a trap** — structural changes must go
  into the template, or the next `theme-switch.sh` overwrites them.

## Confirmed root causes (from audit)

- **Notifications hang forever:** mako has `ignore-timeout=0`, so it *honors* the
  `expire-timeout=0` (persistent) hint that Chrome/YouTube send. `default-timeout=5000`
  only applies when an app sends no timeout — irrelevant for browsers.
- **Wallpaper switching broken:** system has `awww` 0.12.1 (an swww fork), not `swww`.
  `theme-switch.sh:71` and `rofi/scripts/wallpaper.sh:49` still call `swww img`.
  (Autostart correctly uses `awww`.)
- **Ironbar not themed:** hardcoded Nord while system runs catppuccin-mocha; no
  ironbar flavours template; reuses scripts from `waybar/scripts/`.
- **Stale references:** mako `icon-path=/usr/share/icons/Papirus-Dark` (not installed);
  windowrules for pavucontrol/blueman-manager/nm-connection-editor/file-roller (not installed);
  flavours waybar hook `pkill waybar; waybar &` (dead — ironbar is the active bar).
- **Doc drift:** docs claim active theme "Rose Pine" (actual: catppuccin-mocha) and
  describe waybar as the bar. Generated PDFs/HTML committed alongside sources.

## Phase 1 — Mako notifications

**Goal:** notifications auto-dismiss; browser/YouTube can no longer hang; genuinely
critical alerts stay sticky.

Changes (applied to BOTH `~/.config/flavours/templates/mako/templates/default.mustache`
AND the live `~/.config/mako/config`, then `makoctl reload`):

- `ignore-timeout=1` — mako forces its own timeout on every notification, ignoring
  app-sent hints. This is the fix for the YouTube hang.
- `default-timeout=8000` (8s).
- `[urgency=critical]` block overrides back to `ignore-timeout=0` + `default-timeout=0`
  so critical alerts stay until dismissed ("force + keep critical sticky").
- Fix `icon-path` → an installed theme (Breeze, matching `XCURSOR_THEME`, or hicolor).
  Verify the chosen path exists before committing.
- Add `group-by=app-name` so repeated notifications from one app collapse.

**Verification:** trigger a test notification with `notify-send -t 0` (persistent hint)
and confirm it dismisses after 8s; trigger `-u critical` and confirm it stays.

## Phase 3 — Ironbar becomes the one true bar

**Goal:** ironbar fully replaces waybar — themed, self-contained, at feature parity.

1. **Wire into flavours:** create an `ironbar` template producing `~/.config/ironbar/style.css`
   (or a colors include), register it in `flavours/config.toml`, and replace the stale
   `waybar` entry. Ironbar reloads CSS live, so the reload hook is a no-op/touch.
2. **De-couple from waybar:** move borrowed scripts (`popup-toggle.sh`, `colorpicker.sh`,
   weather, `bluetooth-profile-toggle.sh`) into `~/.config/ironbar/scripts/`; repoint
   `config.toml`. Ironbar must not depend on `waybar/` once it's deleted.
3. **Fix broken modules:** triage each module vs waybar parity; fix per-module. Mind the
   GTK4 PUA-icon constraints (BMP PUA range U+E000–U+F8FF only; FiraCode Nerd Font).
4. **Retire waybar:** once at parity, delete `waybar/` config and its flavours template.

**Verification:** switch themes and confirm ironbar restyles; confirm every module works
(workspaces, music, volume, weather, tray, bluetooth, sysinfo, clock, colorpicker, keybinds).

## Phase 4 — Convert to a GNU Stow dotfiles repo

**Decision:** manage with **GNU Stow**, not a bare `git init`. Stow's model is files
*in the repo*, symlinks *out to `~/.config`* — the inverse of the current setup, where
repo dirs are symlinks pointing *into* `~/.config`. So this phase is a restructure.

### Restructure

- Adopt Stow's package layout: each app is a package whose tree mirrors `$HOME`.
  E.g. `hyprland-rice/hypr/.config/hypr/hyprland.conf`, `mako/.config/mako/config`,
  `zsh/.zshrc`. Running `stow hypr` from the repo root symlinks `~/.config/hypr` →
  the repo.
- **Migration is destructive to the current symlinks** and must be done carefully to
  avoid data loss, since the repo dirs currently *are* the symlinks (the real files
  live in `~/.config`). Procedure per package:
  1. Copy the real files from `~/.config/<app>` into the repo at `<app>/.config/<app>/`.
  2. Remove the old backwards symlink and the live `~/.config/<app>`.
  3. `stow <app>` to recreate `~/.config/<app>` as a symlink into the repo.
  4. Verify the app still reads its config (the symlink resolves, app reloads cleanly).
  Do one package at a time; verify before moving on. Keep a backup of `~/.config`
  (e.g. `cp -a ~/.config ~/.config.bak`) until all packages are migrated and verified.
- Confirm `stow` is installed (`pacman -Q stow`); if not, copy the install command to
  clipboard per user's no-sudo rule.

### git + cleanup

- `git init` with a `.gitignore`. **Wallpapers (347MB) stay out of git** and out of any
  Stow package — externally managed, not LFS.
- Delete generated/clutter artifacts: `claude.pdf`, `claude.html`, `claude.md`,
  `project-summary.pdf`, `project-summary.md`, `THEMING.pdf`, `THEMING.md`, `sshot.png`.
- **One minimal `README.md`** only: what the rice is, the Stow install command
  (`stow */` or per-package), and a one-line note per app. Before deleting `PACKAGES.md`,
  mine it for anything genuinely useful (key package list for rebuilds) and fold a
  trimmed version into the README. No LLM-style padding, no per-app deep docs.
- Commit a clean baseline **after** phases 1–3 land, so history starts from a working state.
